db.py Deep Dive MongoDB Collections & Functions
5 MongoDB Collections (like database tables) defined in db.py
sessions
Stores session ID, language, voice emotion, timestamps. Auto-deletes after 90 days (TTL index).
biomarkers
Stores raw voice features: pitch, tone, MFCC mean values + emotion probability scores.
chat_history
Every single chat message (user + assistant) stored with role, stage, and timestamp.
predictions
Final ML prediction results: condition, level (mild/moderate/severe), all input features used.
feedback
Thumbs up/down votes from user on chatbot responses. Stored with reward integer for future RL.
5 Functions in db.py Plain English
1
save_session() Called after voice intro. Saves session ID + detected voice emotion to sessions collection.
2
save_biomarkers() Saves raw pitch/tone/MFCC values + emotion probability scores to biomarkers collection.
3
save_chat_message() Inserts every single message (user or AI) into chat_history with timestamp.
4
save_prediction() Saves the final ML result: condition (anxiety/stress/depression), severity level, all features.
5
save_feedback() Saves thumbs up (+1) / thumbs down (-1) vote to feedback collection for RLHF future use.
ML Training Pipeline From Raw Data to .joblib Model
STEP 1
Raw Data (data/)
Kaggle CSV: Student Mental Health Survey. Contains columns: anxiety score, stress score, depression score, sleep hours, gender, age, academic performance, etc.
→
STEP 2
Preprocessing (preprocessing/)
Python scripts clean data: handle missing values, encode categories (gender → 0/1), normalize numeric columns, split into train/test sets.
→
STEP 3
Training (training/ + notebooks/)
Jupyter notebooks experiment with models. Final scripts train scikit-learn classifiers (Random Forest / SVM / Logistic Regression) on anxiety, stress, depression.
→
STEP 4
Artifacts (artifacts/)
Trained model + preprocessor saved as .joblib files. These are loaded at Flask startup using joblib.load(). No retraining needed at runtime.
The .joblib files in artifacts/ are the "frozen brain" trained once offline, used forever in production
Full User Journey Step by Step (What Happens in Real Time)
1
User opens browser → React app loads (localhost:3000)
React fetches from Flask backend at localhost:5000. User grants microphone permission. MediaRecorder API is initialized in browser. Language selected (English or Urdu).
↓
2
Voice Check-In 30-second recording (VoiceCheckIn component)
User clicks "Start Voice Check-In". Browser MediaRecorder captures up to 30 seconds of audio as a WAV/WebM blob. A timer counts down. Frontend extracts voice biomarkers: pitch (fundamental frequency), tone (spectral centroid), rhythm (speech rate), and MFCC (Mel-frequency cepstral coefficients) using Web Audio API before sending.
↓
3
POST /voice-intro → Flask receives voice features
React sends extracted features via HTTP POST to Flask API. Flask calls save_biomarkers() in db.py to store pitch, tone, MFCC values in MongoDB biomarkers collection. Flask runs a rule-based OR ML-based voice emotion classification: maps voice features to one dominant emotion (calm, anxious, sad, neutral). Calls save_session() with the dominant emotion.
calm anxious sad neutral distressed
↓
4
Questionnaire User answers follow-up mood questions
React shows TherapyCards component with questions: How many hours did you sleep? How stressed do you feel (1-10)? How is your academic/work performance? Are you socially withdrawn? These are standardized GAD-7 / PHQ-9 style screening questions. Answers collected as structured JSON.
↓
5
POST /predict → ML Fusion Model runs
React sends questionnaire answers + voice emotion to Flask. Flask loads artifacts/*.joblib (preprocessor + classifier). Preprocessor transforms answers into numeric feature vector. ML model predicts: condition (anxiety / stress / depression) AND level (mild / moderate / severe). Fusion: voice emotion is blended with survey score to adjust final prediction. Flask calls save_prediction() → stored in MongoDB predictions collection.
anxiety: moderate stress: mild depression: low
↓
6
Therapeutic Chatbot Empathetic conversation begins
Flask returns the prediction + an initial empathetic message. React displays it in the chat UI. User continues chatting. Every user message → POST /chat → Flask builds a response using the prediction context + chat history. Rule-based OR LLM-powered responses suggest coping exercises (breathing, journaling, grounding). Every message pair saved via save_chat_message() to MongoDB.
↓
7
Text-to-Speech (Edge-TTS / gTTS) Bot speaks the response
Flask calls Edge-TTS or gTTS (from requirements.txt). Converts chatbot text response to audio file (MP3/WAV). Supports English AND Urdu voices. Flask returns the audio file URL or streams it. React plays audio automatically in the browser. Bilingual support: if user selected Urdu, TTS uses Urdu voice model.
↓
8
Emotion Trend Chart Visual progress tracking (Recharts)
React renders emotion tracking charts using the Recharts library. Line/bar chart shows emotion scores over time. Data pulled from MongoDB chat history or predictions collection. User can see their mood improving or changing over multiple sessions.
↓
9
Feedback Thumbs up / down on chatbot response
User rates each chatbot message. React sends vote to Flask POST /feedback. Flask calls save_feedback() → stored in MongoDB feedback collection with reward (+1 or -1). This data can be used for future Reinforcement Learning from Human Feedback (RLHF) to improve the chatbot.
Flask API Endpoints (backend/app.py) What Each Route Does
POST
/voice-intro
Receives voice biomarkers (pitch, tone, MFCC). Detects dominant emotion. Saves session + biomarkers to MongoDB. Returns emotion label + confidence.
POST
/predict
Receives questionnaire answers + voice emotion. Runs joblib ML pipeline. Returns condition + severity level. Saves prediction to MongoDB.
POST
/chat
Receives user message + session ID. Generates empathetic chatbot response with coping suggestions. Saves both messages to chat_history. Returns reply text.
GET/POST
/tts
Receives text + language. Calls Edge-TTS or gTTS. Returns audio file (MP3) for browser playback. Bilingual: English + Urdu voices.
POST
/feedback
Receives thumbs up/down vote + message index. Saves to feedback collection with +1/-1 reward for future RLHF training.
GET
/history
Returns emotion history from MongoDB for the current session. Used by React Recharts to render trend graphs.
Complete Tech Stack Every Library and Why It's There
React.js
The entire frontend UI. Component-based. VoiceCheckIn, TherapyCards, Chat, Charts are all React components.
Recharts
React charting library. Draws the emotion trend graphs (line charts, bar charts) on the dashboard.
Flask
Python web framework. Runs the backend server. Handles all API routes. Lightweight and perfect for ML APIs.
flask-cors
Allows React (port 3000) to talk to Flask (port 5000). Without CORS, browser blocks cross-origin requests.
scikit-learn
ML library. Trains Random Forest/SVM/Logistic Regression models for anxiety, stress, depression classification.
joblib
Saves and loads trained models to/from .joblib files. joblib.dump() after training, joblib.load() in Flask.
numpy
Numerical arrays. Used for MFCC processing, feature vectors, and ML preprocessing math.
pandas
Data manipulation. Reads CSV datasets, cleans data, creates DataFrames for model training.
gTTS
Google Text-to-Speech. Converts chatbot text to MP3 audio. Supports multiple languages including Urdu.
Edge-TTS
Microsoft Edge TTS API. Higher quality bilingual speech synthesis. Used for English + Urdu voice output.
MongoDB
NoSQL database (Atlas cloud). Stores sessions, biomarkers, chat history, predictions, feedback. Connected via pymongo.
pymongo
Python driver for MongoDB. Used in db.py to connect and run insert/find operations on collections.
python-dotenv
Loads secrets from .env file: MONGO_URI, API keys. Keeps sensitive data out of source code.
Docker
Packages the app into a container so it runs the same anywhere. Dockerfile at root defines the build.
Jupyter
Used in notebooks/ for exploratory data analysis and model experiments before writing production scripts.
The Complete Flow in One Paragraph Perfect for Viva
The user opens the React frontend → records 30 seconds of voice → browser extracts pitch, tone, and MFCC features → sends to Flask /voice-intro → Flask detects dominant emotion and saves to MongoDB via db.py → user answers a mood questionnaire → Flask /predict loads the .joblib ML model from artifacts/ → fuses voice emotion + survey scores → predicts anxiety/stress/depression level → saves prediction to MongoDB → chatbot begins a therapeutic conversation with coping suggestions → Flask /tts converts each reply to audio (English or Urdu) → React plays it → user sees emotion trend charts → votes 👍 or 👎 → feedback saved to MongoDB for future RLHF improvement.
